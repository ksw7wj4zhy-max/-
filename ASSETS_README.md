## 替换规则

### 替换素材图片
1. 准备好 11 张 PNG 图片（建议带透明背景）
2. 命名为 `fruit_00.png` ~ `fruit_10.png`
3. 放入 `assets/` 文件夹覆盖即可
4. 游戏会自动加载

### 修改游戏参数
在 `index.html` 中找到 `GRADES` 数组：
```javascript
const GRADES = [
  {emoji:'🐟', r:18, score:1,   color:'#ff0040', rgb:[255,0,64]},   // 等级0
  // ... r=半径(px), score=合成得分, color=粒子/备选颜色
  {emoji:'🐉', r:82, score:1024, color:'#15803d', rgb:[21,128,61]}, // 等级10
];
```

## 核心逻辑修复记录
- ✅ **等级10碰撞 = 胜利**：两个最高级水果（fruit_10.png）碰撞时触发胜利结算页面
- ✅ 死亡线稳定期检测：水果必须完全落定后才触发游戏结束
- ✅ 碰撞物理4轮迭代：避免水果穿透
- ✅ 合并无敌时间：60帧内不会误触发
- ✅ 非正方形图片自适应缩放：保持比例填充

## 胜利条件
当两个 **等级10（fruit_10.png）** 的水果碰撞并合成时：
- 游戏胜利
- 展示「合成成功！」结算页面
- 显示最终得分
- 提供「再玩一次」按钮
